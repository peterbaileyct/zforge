# Z-World
A Z-Forge World specification is a Z-Bundle used for [RAG](RAG%20and%20GRAG%20Implementation.md). The type slug for it is "world". It contains the following elements:

## Raw Text
The original input text will be kept so that the other elements can be regenerated by later versions of the app as new features are added, e.g. new node types to look for and a more detailed ETL process.

## Key-Value
- Title (human-readable, full text, e.g. "The Dragonet Prophecy")
- slug (dash-case, unique among Z-Worlds on the same device, e.g. "lord-of-the-rings")
- UUID
- Summary (1-3 paragraphs of plain text describing the world in diegetic terms, used to help unfamiliar users to pick a world for experience generation and to help kick-start experience generation, especially where no prompt is given)
- Setting era (brief label for when the world is nominally set, e.g. "pre-industrial fantasy", "far future", "alternate 1920s"; used as a quick LLM hint during experience generation)
- Source canon (list of source work titles — books, films, games — the world is drawn from; used for disambiguation between continuities and to guide the document parsing pipeline)
- Content advisories (list of thematic flags relevant to experience generation, e.g. "moderate violence", "political intrigue", "body horror"; informs tone guidance and content guardrails)

# ============================================================
# WORLD BIBLE ONTOLOGY
# ============================================================
#
# `agent` and `ENTITY` in relationship declarations below are
# macro-expansions, not extractable node types:
#   agent  → character, species, organization
#   ENTITY → all node types
# Neither appears in allowed_nodes or generates a KuzuDB node table.


# ============================================================
# ENTITIES
# ============================================================

# --- Agent Umbrella ---

agent: Macro expansion only — not a KuzuDB node type. In relationship
  declarations, expands to: character, species, organization.

character: A named sapient individual of narrative importance —
  protagonist, antagonist, NPC, historical figure, or unreliable
  narrator. Includes living, dead, undead, ascended, divine, and
  synthetic individuals.
  Classifier flags: is_divine (bool), is_synthetic (bool)

species: A biological, metaphysical, or constructed category of being —
  race, creature type, magical lineage, or manufactured form. Defines
  innate traits and capabilities shared by all instances. Covers
  non-sapient creature types as well as sapient races.
  Classifier flags: is_sapient (bool)

organization: A named group of agents united by shared structure or
  purpose — guild, government, army, noble house, criminal network,
  trade company, political party, or informal faction.


# --- Place ---

location: Any place of narrative importance regardless of scale — from
  a single room up to a city, moon, or pocket dimension. Includes
  constructed places such as buildings, dungeons, and vessels.
  Classifier flags: is_constructed (bool)

region: A large-scale named territory that contains locations — a
  continent, kingdom, biome, plane of existence, or star system.
  Regions are traversed rather than visited directly.


# --- Object ---

item: A discrete physical object of narrative significance — weapon,
  artifact, relic, tome, crown, or MacGuffin. Includes consumable
  resources when they are individually named and narratively significant.
  Classifier flags: is_consumable (bool)


# --- Time ---

event: A discrete occurrence with narrative weight — battle, treaty,
  disaster, ritual, coronation, prophecy, birth, death, or discovery.

time_period: Any named or bounded span of time — a war, reign, age,
  dynasty, or epoch. May be defined by calendar dates, bracketing
  events, or cultural consensus. Point-like time markers (specific
  years, named days) are modelled as string properties on event and
  time_period nodes rather than standalone nodes.


# --- Attributes ---
#
# Traits are modelled as a `traits` string property on entity nodes
# (e.g. character.traits = "cynical, honourable, haunted") rather than
# standalone nodes. This avoids deduplication ambiguity (is "cynical"
# one shared node or one per character?) and keeps trait data queryable
# via WHERE n.traits CONTAINS '...' without a separate node table.


# --- Culture & Belief ---

culture: A named set of shared practices, norms, aesthetics, and
  values held by a group. Languages and economies are modelled as
  string properties of culture or organization rather than
  standalone nodes.

belief_system: A religion, philosophy, code of ethics, magical
  tradition, or ideology that shapes behaviour. Covers organised
  religions and magic systems as subtypes.
  Classifier flags: type (religion/philosophy/magic_system/ideology/code)

law: A named rule, statute, or system of governance that carries
  enforcement — edicts, accords, constitutions, or metaphysical laws
  of nature. Distinct from belief_system in that it implies
  enforcement rather than voluntary adherence.

myth: A widely held story, legend, or belief that may or may not be
  literally true. Valuable for tracking unreliable world history and
  distinguishing fact from legend at generation time.


# --- Narrative Structure ---

conflict: A named, persistent opposition between agents or forces —
  a war, political rivalry, personal vendetta, or philosophical schism.

goal: A desired outcome actively pursued by one or more agents —
  "reclaim the throne", "destroy all magic", "survive the winter".
  Goals are first-class nodes because they can be shared across agents
  (enabling "who else wants this?") and because pursuit is time-limited
  via from_time/to_time on the AGENT wants GOAL edge, not on the node.

## Implementation

The following are the authoritative values passed by the [World Generation](World%20Generation.md) process to the [Document Parsing pipeline](Parsing%20Documents%20to%20Z-Bundles.md).

### Macro Expansion

The `AGENT` and `ENTITY` shorthand used in the relationship declarations above must be expanded to concrete node type lists before calling `_pre_create_schema` and `LLMGraphTransformer`:

| Macro | Expands to |
|---|---|
| `AGENT` | `Character`, `Species`, `Organization` |
| `ENTITY` | all `allowed_nodes` entries |

### `allowed_nodes`

`Character`, `Species`, `Organization`, `Location`, `Region`, `Item`, `Event`, `TimePeriod`, `Culture`, `BeliefSystem`, `Law`, `Myth`, `Conflict`, `Goal`

### `allowed_relationships`

`is_a`, `instance_of`, `subtype_of`, `lives_in`, `located_in`, `originates_from`, `born_in`, `died_at`, `contains`, `nested_in`, `adjacent_to`, `native_to`, `occurs_within`, `born_during`, `died_during`, `overlaps`, `preceded_by`, `named_after`, `owns`, `controls`, `rules`, `created_by`, `knows`, `related_to`, `descended_from`, `allied_with`, `enemy_of`, `loyal_to`, `betrayed`, `mentored`, `loves`, `fears`, `member_of`, `employed_by`, `opposes`, `operates_through`, `founded_by`, `based_in`, `participated_in`, `witnessed`, `caused`, `triggered_by`, `resulted_in`, `occurred_at`, `founded`, `created`, `destroyed`, `affected_by`, `wants`, `targets`, `seeks`, `believes_in`, `follows`, `governed_by`, `belongs_to`, `practices`, `practised_in`, `knows_about`, `hides`, `reveals`, `misinformed_about`, `symbol_of`, `part_of`, `originated_in`, `conflicts_with`, `derived_from`, `governs`, `embodies`, `believed_by`, `about`, `involves`, `between`

### `relationship_properties`

These are the edge property names passed to `LLMGraphTransformer` for extraction. They are chosen because they are most likely to be explicitly stated in prose source text:

| Property | KuzuDB column type | Notes |
|---|---|---|
| `from_time` | `STRING` | Temporal marker when relationship began |
| `to_time` | `STRING` | Temporal marker when relationship ended |
| `role` | `STRING` | Role held by the source entity in this relationship |
| `perspective` | `STRING` | Agent id whose viewpoint this reflects, if subjective |
| `canonical` | `BOOL` | `false` for rumour/legend/myth; defaults to `true` |

- `from_time` / `to_time` — extracted from temporal markers in the text ("since the coup", "until Year 1045", "during the reign of Vetinari")
- `role` — extracted for participation and membership relationships ("as Commander", "acting as spy", "in the role of founder")
- `perspective` — extracted when a statement is attributed to a viewpoint ("according to dwarfish tradition", "in Vimes's view")
- `canonical` — set to `false` when the text signals uncertainty or legend ("reportedly", "it is said", "legend holds that"); defaults to `true`

All other fields in the [Universal Metadata Schema](#universal-metadata-schema) (`certainty`, `intensity`, `context`, `contradiction_group`, and relationship-specific qualifiers) are **not** included in `relationship_properties` — they cannot be reliably extracted from prose and are reserved for future manual curation or a dedicated enrichment pass.

### `node_properties`

These are the node property names passed to `LLMGraphTransformer` for extraction alongside each entity:

`traits`, `is_divine`, `is_synthetic`, `is_sapient`

- `traits` (`STRING`) — comma-separated list of persistent characteristics, extracted for `Character` nodes (e.g. `"cynical, honourable, haunted"`). Queryable via `WHERE n.traits CONTAINS '...'`.
- `is_divine` (`BOOL`) — `true` for characters who are gods, demons, or ascended beings; extracted for `Character`.
- `is_synthetic` (`BOOL`) — `true` for constructed intelligences, golems, androids; extracted for `Character`.
- `is_sapient` (`BOOL`) — `true` for sapient species; `false` for non-sapient creature types; extracted for `Species`.

### Node Column Definitions

All node tables are created with `id STRING, type STRING` as minimum columns. The following types carry additional columns declared in `_pre_create_schema`:

| Node type | Additional columns |
|---|---|
| `Character` | `traits STRING, is_divine BOOL, is_synthetic BOOL` |
| `Species` | `is_sapient BOOL` |
| `Chunk` | `text STRING` (replaces type — see RAG spec) |

All other node types use only the minimum `id STRING, type STRING` schema.

### `entity_type` Casing (LanceDB ↔ KuzuDB)

The `entity_type` column in LanceDB uses **snake_case**; KuzuDB node table names use **PascalCase**. The mapping is deterministic (PascalCase → snake_case), but the multi-word types must be stated explicitly to avoid ambiguity:
